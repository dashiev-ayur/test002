# Что сделано: фаза 5

Документ фиксирует UI MVP после фазы **5** (один экран настроек Telegram). Обмен данными с сервером — по контрактам из [ТЗ.md](./ТЗ.md); стек и архитектура UI — [Frontend.md](./Frontend.md), порядок фаз — [План реализации MVP.md](./План%20реализации%20MVP.md).

---

## Реализовано

- Каталог **`frontend/`**: SPA на Vite, React, TypeScript.
- Экран по маршруту из ТЗ: **`/shops/:shopId/growth/telegram`** (форма: токен, chat_id, тумблер, сохранение; блок статуса; текстовая подсказка про chat_id).
- Запросы к API с настраиваемым базовым URL через **`VITE_API_BASE_URL`** (пустое значение — тот же origin, что и у фронта). Пример — в `frontend/.env.example`.
- Слои по FSD (`app`, `pages`, `widgets`, `entities`, `shared`), зависимости и практики — как в [Frontend.md](./Frontend.md).

---

## Команды

Из каталога `frontend/`:

```bash
npm install
npm run dev      # разработка
npm run build    # сборка
npm run lint     # ESLint
```
