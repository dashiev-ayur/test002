# HTTP API

Все методы отдают **JSON**. Заголовок запроса с телом: `Content-Type: application/json`.

**Префикс:** пути ниже относительно базового URL приложения (например `http://localhost:8000`).

**Параметр пути `shopId`:** целое число (`\d+`).

Для запросов с другого origin настроен CORS на префикс `/shops/…` (см. `CORS_ALLOW_ORIGIN` в `.env`). Поддерживаются методы `GET`, `POST`, `OPTIONS`.

---

## POST `/shops/{shopId}/telegram/connect`

Подключение или обновление настроек Telegram для магазина (upsert по `shopId`).

### Тело запроса

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `botToken` | string | да | Токен бота (после JSON выполняется trim; не может быть пустым) |
| `chatId` | string | да | Идентификатор чата (trim; не пустой) |
| `enabled` | boolean | да | Включена ли отправка уведомлений |

Пример:

```json
{
  "botToken": "123456:ABC-DEF...",
  "chatId": "987654321",
  "enabled": true
}
```

### Успешный ответ

**Код:** `200`

```json
{
  "id": 1,
  "enabled": true,
  "botToken": "****xxxx",
  "chatId": "98******21",
  "createdAt": "2026-04-04T12:00:00+00:00",
  "updatedAt": "2026-04-04T12:00:00+00:00"
}
```

Даты в формате ISO 8601 (ATOM). Поля `botToken` и `chatId` в ответе **маскируются**.

### Ошибки

| Код | Тело | Когда |
|-----|------|--------|
| `404` | `{"error": "…"}` | Магазин с данным `shopId` не найден |
| `422` | `{"error": "…"}` | Некорректный JSON, пустое тело, или пустые `botToken`/`chatId` после trim |
| `422` | `{"errors": {"поле": ["сообщение"]}}` | Нарушение правил Validator (например отсутствует обязательное поле) |

## POST `/shops/{shopId}/orders`

Создание заказа (эмуляция). Заказ сохраняется в БД; при включённой интеграции выполняется попытка уведомления в Telegram (ошибка Telegram **не** отменяет созданный заказ).

### Тело запроса

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `number` | string | да | Номер заказа (trim) |
| `total` | number или string | да | Сумма (число или строка с числом; после десериализации строка дополнительно trim) |
| `customerName` | string \| null | нет | Имя клиента (trim; пустая строка приводится к отсутствию значения) |

Пример:

```json
{
  "number": "A-1005",
  "total": 2490,
  "customerName": "Анна"
}
```

### Успешный ответ

**Код:** `201 Created`

```json
{
  "order": {
    "id": 1,
    "number": "A-1005",
    "total": "2490.00",
    "customerName": "Анна",
    "createdAt": "2026-04-04T12:00:00+00:00"
  },
  "notificationStatus": "sent"
}
```

Поле `notificationStatus` — результат попытки уведомления для **этого** созданного заказа:

- `sent` — сообщение успешно отправлено, в журнал записан успех;
- `failed` — ошибка Telegram, заказ создан, в журнал записана неудача;
- `skipped` — отправка не выполнялась (интеграции нет / выключена / для пары магазин+заказ запись в журнале уже есть / конфликт идемпотентности при записи журнала).

### Ошибки

| Код | Тело | Когда |
|-----|------|--------|
| `404` | `{"error": "…"}` | Магазин не найден |
| `422` | `{"error": "…"}` | Некорректный JSON, пустое тело |
| `422` | `{"errors": {…}}` | Нарушение валидации (`number`, `total` и т.д.) |

## GET `/shops/{shopId}/telegram/status`

Агрегированный статус интеграции и отправок за последние **7 суток**.

### Успешный ответ

**Код:** `200`

```json
{
  "enabled": true,
  "chatId": "98******21",
  "lastSentAt": "2026-04-04T11:00:00+00:00",
  "sentCount": 3,
  "failedCount": 0
}
```

| Поле | Описание |
|------|----------|
| `enabled` | `true`, если есть интеграция и она включена; иначе `false` |
| `chatId` | Маскированный идентификатор чата или **`null`**, если записи интеграции ещё нет |
| `lastSentAt` | Время **последней успешной** отправки (`SENT`) или **`null`** |
| `sentCount` | Число успешных отправок за 7 дней |
| `failedCount` | Число неуспешных попыток за 7 дней |

### Ошибки

| Код | Тело | Когда |
|-----|------|--------|
| `404` | `{"error": "…"}` | Магазин не найден |
